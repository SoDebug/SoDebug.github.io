这里以最简单的Flask应用为例，对其进行逐步分析

一、 分析最简单的Flask应用

按照Official Docs的指引，上述代码意思如下：

  • from flask import Flask ：导入 Flask 类，该类的实例将会成为WSGI 应用
  • app = Flask(__name__)：创建Flask的实例——app。第一个参数（这里是__name__）是应用模块的名称， __name__ 是一个适用于大多数情况的快捷方式。正是由于这个参数， Flask 才能知道在哪里可以找到模板和静态文件等东西。
  • @app.route('/')：route()是装饰器，用于告知Flask触发函数的URL。
  • return "<p>Hello, World!</p>" ：函数返回的内容是在用户浏览器中显示的信息。默认的内容类型是 HTML ，因此字 符串中的 HTML 会被浏览器渲染。

除了请求对象之外还有一种称为 session 的对象，允许在不同请求 之间储存信息。

这个对象相当于用密钥签名加密的 cookie ，即用户可以查看 cookie ，但是如果没有密钥就无法修改它。

注意：使用会话之前您必须设置一个密钥。

当用户请求应用程序的根目录路由 ‘/‘ 时，index() 函数会被调用。在函数中, 使用 session 字典来检查用户是否已经登录。如果 session 中包含 ‘username’ 键，则说明用户已经登录，函数会返回一个包含用户名的字符串。否则，函数会返回一个提示用户未登录的字符串。

当用户请求登录路由 ‘/login’ 时，login() 函数会被调用。如果请求方法是 ‘POST’，说明用户提交了登录表单。在这种情况下,函数会使用 request.form 字典来获取用户名,并将其存储在 session[‘username’] 中。然后,使用 redirect() 函数重定向到根目录路由 ‘/‘。如果请求方法是 ‘GET’，函数会返回一个包含登录表单的 HTML 代码的字符串。

当用户请求注销路由 ‘/logout’ 时，logout() 函数会被调用。函数会使用 session.pop() 方法删除 session 中的 ‘username’ 键,并重定向到根目录路由 ‘/‘。这样会使用户退出登录。
