一、Flask与Django分别是什么？

1.1 Flask 是什么？

Flask以Python为逻辑语言，以 Werkzeug WSGI Tool Package、 Jinja2 Engine作为核心引擎，用于构建Web Application Framework应用程序的轻量级框架。

1.2 Django 是什么？

Django 以Python为逻辑语言，以 Model-View-Controller (MVC) 架构为核心构架，主攻用于快速开发安全、可靠和可维护的 Web Application Framework重量级框架。

二、 相对于Django而言，Flask 的优点是什么？

2.1 Flask更简单易用

Flask 框架的核心部分非常简单，易于理解和使用，适合入门和小型项目。

Flask 框架的核心部分非常简单，主要是因为它没有很多内置功能，它只是一个简单的 web 框架，主要提供了如下功能：

  • 路由系统：用于定义视图函数的 URL 路径。
  • 模板系统：用于渲染 HTML 模板。
  • 错误处理：用于处理应用程序中出现的错误。
  • 请求对象和响应对象：用于处理 HTTP 请求和响应。

这些功能都是基本的 web 框架功能，Flask的设计目的就是让这些功能尽可能简单，易于理解和使用。

与之对比，Django是一个功能更加丰富的框架，它包括了很多内置的功能，如：

  • 管理数据库的ORM系统
  • 表单验证和身份验证
  • 管理后台
  • 基于类的视图
  • ……

2.2 Flask灵活性更高

Flask 提供了很多扩展，你可以根据需要选择性地使用，没有多余的功能限制。（社区提供）

Flask灵活性高，主要是因为它设计成模块化，核心部分非常简单，大部分功能是通过扩展来实现的。这样一来，开发者可以根据自己项目的需求来选择使用哪些扩展。

Flask扩展可以为应用程序增加很多实用的功能，如：

  • 管理数据库的ORM系统
  • 表单验证和身份验证
  • 管理后台
  • 电子邮件
  • 图像处理
  • ……

可以根据项目需求来选择使用哪些扩展，没有多余的功能限制你。

而与之相比，Django是一个功能更加丰富的框架,它自带很多功能，可能会因为不需要而造成学习负担或者资源浪费。

这种灵活性能够使开发者更灵活地选择所需要的功能，不会因为不需要而增加学习成本和资源浪费。

2.3 Flask开发周期更短

开发周期短：由于 Flask 框架的简单性，可以让开发人员更快地完成项目。

三、 相对于Django而言，Flask 的缺点是什么？

3.1 Flask功能有限、扩展性差、不适合做大项目

Flask 是一个轻量级框架，其内置功能有限，需要使用第三方库才能完成一些常见功能

直白的说，Flask的核心框架简单的优点也是它的缺点。例如管理数据库,表单验证等,如果需要这些功能需要手动添加。

四、最终选择

大致考虑一些，Flask框架更适合做这种非长期维护型项目。主要是不需要考虑安全性、稳定性等维护支持。
