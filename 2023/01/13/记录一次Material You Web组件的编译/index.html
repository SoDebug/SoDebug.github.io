一直想着自己制作一款Material You风格的产品（App、Web），这次假借毕业设计之名，想要设计出一款Material You风格的教务系统

一、官方技术支持正在进行时？

既然决定去做了，就应当师承正统

Material You出自Google旗下，这里就直接去它的官网找了。从官方的web demo上看效果确实不错。

但是有一个问题，Google官方对Material You Web平台的开发并没有完成。公告上显示：

Web support is planned

The Material Web team is making significant changes to the codebase in preparation for adding Material Design 3 support. Follow the material-web repository on Github for updates.

二、Github repo 尝鲜？

正如上段所说，官方技术支持正在进行时，我们便根据官方文档的指引找到了它的源代码库

看到官方任然正在开发时本准备无功而返了，但是我在issues里面发现了Google的计划发布日期相关的issue:

这里说到，Google Material 3 Web 相关技术团队预备在2022年底前发布第一个版本。而今是：2023/01/13，那么，此时官方应该有一个基本完备的版本了。

然后在仓库的Release页面发现了最新的版本：v1.0.0-pre.1

可以看出这是与发布版本，不出意外，这个版本和正式版相比不会有太大的变化。因此我准备就这个版本进行尝鲜。

三、使用方法？

看了一下本repo的REDAME.md，发现官方并没有为其准备使用指南。

=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=><=

然后我想的是既然都是一家开发的，那么Material 2的使用指南会不会能用。

官方README里面提到：Note: Looking for Material 2? MWC components are now on the mwc branch.

也就是原MD2，现在同样在本repo中，其为mwc分支。

四、编译环境部署

4.1 Windows 部署

4.1.1 第一次部署

电脑环境如下：

items envs
Windows Version 22623.1095
Node Version 16.18.1
  • 克隆仓库源码到本地：git clone https://github.com/material-components/material-web.git
  • 尝试安装：npm install

然后就是报错：

从目前的报错情况中，它要我们在执行npm install前需要先配置以下环境：

  • Visual Studio (including the “Desktop development with C++” workload)

然后我就去下载了最新的Visual Code 2022，并按照它的要求安装Desktop development with C++。

4.1.2 第二次部署

items envs
Windows Version 22623.1095
Node Version 16.18.1
Visual Code Version 2022
  • 继续安装：npm install

然后又是报错：

这次的报错情况中，它要我们在执行npm install前需要先配置要先配置全局Python环境，然后因为我是使用Conda Mini配置的环境。

然后我就把该Python环境添加到系统环境变量了。

4.1.3 第三次部署

items envs
Windows Version 22623.1095
Node Version 16.18.1
Visual Code Version 2022
Python Version 3.8.15

现在继续执行npm install。

然后又是报错：

这次报错是说缺少node-gyp，这次直接去node-gyp仓库找办法安装，gyp给出的安装办法是：npm install -g node-gyp

4.1.3 第四次部署

items envs
Windows Version 22623.1095
Node Version 16.18.1
Visual Code Version 2022
Python Version 3.8.15
node-gpy Version 9.3.1

然后继续安装：npm install

这次npm install顺利，直接进入到下一个报错。

  • npm run build报错:

这个错误消息表明，系统上找不到命令 ‘grep’，这是因为在 Windows 系统上并不带有 grep 命令。

然后我们直接：npm install grep-cli，解决了这个错误。

4.1.4 第n次部署

items envs
Windows Version 22623.1095
Node Version 16.18.1
Visual Code Version 2022
Python Version 3.8.15
node-gpy Version 9.3.1
grep-cli Version lastest

同样的：

  • npm run build报错:

错误原因，系统中缺少 xargs 命令，导致运行失败。但是Windows中没有可以官方安装 xargs 的办法。
